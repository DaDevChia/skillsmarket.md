# Human-Computer Interaction Records

This folder preserves sanitized product-development records for SkillsMarket.

## Purpose

The records document how the interface evolved from a compressed dashboard into a multi-page skills-market analysis product. They are intended for hackathon judging, design review, and future product iteration.

## Sanitization policy

The records intentionally remove or generalize sensitive information:

- No API keys, tokens, credentials, or `.env` values.
- No private email addresses.
- No raw uploaded resumes or user documents.
- No hidden infrastructure identifiers.
- No full unredacted transcript dumps.
- Public product concepts, UX feedback, technical decisions, and non-sensitive source names are preserved.

## Files

- `conversation-record.md` — sanitized chronological record of the human/AI design conversation.
- `design-decisions.md` — UX decisions and rationale.
- `agentic-development-workflow.md` — how Hermes, tmux, Claude Code, Codex, SSH/Termius-style access, and `/goal` prompts were used as the build workflow.
- `supporting-setup-chats.md` — sanitized summaries of the Codex delegation/auth setup and Cloudflare Tunnel auth setup chats.
- `privacy-and-redactions.md` — what was removed and why.
